# AOP 行政运营管理平台 — 产品设计规范

**版本**: v1.0  
**日期**: 2026-07-07  
**架构方案**: Linear 风格单列式极简架构（方案 A）  
**项目类型**: 全新构建（独立于现有行政周报看板）

---

## 一、设计理念

### 核心原则

1. **Single Source of Truth** — 所有页面数据均来源于 Work Item，员工不填写周报，只更新事项
2. **Minimal Input, Max Output** — 员工每周唯一动作：更新自己负责的事项
3. **Push Over Pull** — 「我的工作」页主动提醒待更新事项，无需员工自行寻找
4. **Type-Driven Templates** — 专项和日常运营使用不同编辑/展示模板
5. **Role-Based Views** — 不同角色看到不同视图，数据来源一致
6. **Linear-Style Aesthetics** — 极简、现代、高信息密度、白色背景、蓝色主色

### 视觉规范

- **背景**: 白色 #FFFFFF / 浅灰 #F8F9FB
- **分区**: #F1F3F8
- **主色**: #3B5BDB（蓝色）
- **成功**: #10B981 / **警告**: #F59E0B / **危险**: #EF4444
- **紫色**: #8B5CF6
- **字体**: Inter + Noto Sans SC
- **圆角**: 10-14px / **卡片阴影**: 极轻
- **留白**: 充足 / **边框**: 弱化
- **禁止**: 传统OA、ERP、Windows后台风格

## 二、信息架构

```
AOP 行政运营管理平台
├── 🏠 我的工作      — 员工唯一更新入口（默认首页）
├── 📋 行政总览      — 按业务模块展示
├── ⭐ 重点专项      — type=专项 的筛选视图
├── 📊 运营看板      — 管理层首页（统计+异常）
├── 👥 周会视图      — 按人聚合汇报（最重点）
├── 📝 周报         — 自动生成+可编辑+导出
└── 📈 数据驾驶舱    — KPI Dashboard
```

**数据流向**: 员工更新 Work Item → 系统自动聚合 → 各页面（按人/按模块/按统计/按文档/按KPI）

| 字段 | 类型 | 说明 |
|------|------|------|

| 下周计划 | richtext | 核心字段 |

| 风险 | text | 仅专项 |
| 本周数据 | kpi map | 仅日常运营 |
| 异常 | text | 仅日常运营 |
| 优化措施 | richtext | 仅日常运营 |
| 评论 | list | 通用 |
| 附件 | list | 通用 |

## 五、页面设计详细说明

### 5.1 我的工作（员工首页，默认入口）

**布局**: 全宽卡片列表  
**顶部**:
- 欢迎语 + 本周待更新数/今日截止数
- 4个统计卡片：总事项/待更新/今日截止/延期

**Tab 切换**: 全部 | 待更新 | 已更新 | 延期  
**搜索**: 支持按事项名称搜索  
**排序**: 按优先级分组（🔴 高 → 🟡 中 → 🟢 低），同优先级内按截止时间升序（越紧急越靠前）  
**卡片信息**: 名称 / 模块标签 / 类型标签 / 优先级徽章 / 状态 / 负责人头像 / 截止时间 / 更新状态标记  
**视觉标记**: 左侧橙色边线=本周未更新，红色边线=延期  
**交互**: 点击卡片 → 右侧 Slide-over 展开事项详情

### 5.2 事项详情（Slide-over 面板）

**宽度**: 520px 从右侧滑出  
**专项模板**: 完成率环 / 里程碑列表 / 风险提示 / 本周更新 / 下周计划  
**日常运营模板**: KPI 数据卡片 / 异常提醒 / 优化措施 / 下周重点  
**通用**: 评论列表 + 附件列表  
**底部**: 保存按钮（sticky footer）  
**编辑**: 所有文本字段点击即可编辑（inline edit）

### 5.3 行政总览

**顶部**: 模块标签栏（三组，含子模块，每个标签显示负责人）  
**视图切换**: 列表 ↔ 看板  
**列表视图**: 表格含 事项名称/类型/负责人/状态/完成率/最后更新  
**看板视图**: 按状态分列，卡片含名称/类型/负责人/优先级/完成率  
**交互**: 点击标签切换模块，点击事项打开 Slide-over

### 5.4 重点专项

**自动筛选**: type=专项  
**三种视图**: 甘特图 | 时间轴 | 列表  
**甘特图**: 横轴月/周，条形=项目周期，填充=完成率，菱形=里程碑，红色虚线=今天  
**时间轴**: 纵向时间线，按模块分组，显示阶段和风险  
**列表**: 表格含风险等级徽章和进度条

### 5.5 运营看板（管理层首页）

**统计行**: 5个卡片（总事项/进行中/已完成/延期/高风险）  
**左栏**: 模块分布柱状图 / 负责人工作量排行 / 状态环形图  
**右栏**: 最近更新列表 / 超期事项（红色）/ 需协调事项（紫色）

### 5.6 周会视图（最重要页面）

**左侧面板** (240px): 负责人列表，显示待更新/延期数量 badge  
**右侧面板**: 该负责人所有事项卡片，按优先级排序  
**专项卡片**: 名称/状态/完成率环/本周更新/风险/下周计划  
**日常运营卡片**: 名称/状态/KPI摘要/异常/下周重点  
**两种演示模式**:
- 📺 投屏演示：全屏，放大字号，简化边栏，适合会议投屏
- 📑 PPT翻页：每人一页，底部导航（← 2/9 →），支持键盘翻页

### 5.7 周报（自动生成）

**周选择**: ← 第27周 6月30日-7月4日 →  
**生成信息**: 自动生成于 7月4日 18:00 + 重新生成按钮  
**导出**: PDF / Word / Excel  
**五段结构**:
1. ✅ 本周重点完成（status=已完成的事项）
2. ⭐ 专项项目进展（type=专项，status≠已完成）
3. 🔄 日常运营数据（type=日常运营，含KPI数值）
4. ⚠️ 延期与风险（红色高亮）
5. 🎯 下周重点（聚合所有事项的下周计划）
**编辑**: 每个板块支持手动编辑修改

### 5.8 数据驾驶舱

**行政年度预算**: 总额/已支出/执行率/剩余 + 月度柱状图  
**差旅费用**: 用车/机票/火车/酒店 四维度（数据来源：差旅平台，后续接口同步）  
**项目预算执行率**: 表格含进度条，每项预算 vs 实际采购  
**活动参与率**: 环形图 + 支持手动编辑输入  
**会议室利用率**: 横向柱状图  
**行政服务满意度**: 综合评分 + 分维度评分（支持手动编辑输入）
