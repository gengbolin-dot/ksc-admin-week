Foundation

设计原则

🎯

Single Source of Truth

所有页面数据均来源于 Work Item。员工不填写周报，只更新事项。系统自动生成周报、周会汇报、总览和看板。

⚡

Minimal Input, Max Output

员工每周唯一动作：更新自己负责的事项。系统从更新中自动聚合出所有管理层视图。

🔄

Push Over Pull

系统主动推送待更新事项给员工（更新中心），而非让员工自己去寻找需要更新的内容。

🧩

Type-Driven Templates

专项和日常运营使用不同的编辑模板和展示卡片，同一数据模型，差异化体验。

👁️

Role-Based Views

不同角色看到不同视图，但数据来源一致。员工→我的工作，主管→组视图，负责人→全视图。

✨

Linear-Style Aesthetics

极简、现代、高信息密度。白色背景、浅灰分区、蓝色主色、大量圆角和留白。禁止传统OA/ERP风格。

Architecture

信息架构（Information Architecture）

整个平台以 Work Item 为唯一数据核心，所有页面均为该数据的**不同投影**：

🏗️ AOP 行政运营管理平台
🏠 我的工作 — 员工唯一更新入口
待更新事项卡片列表 / 事项详情编辑 / 更新向导入口
📋 行政总览 — 按业务模块展示
模块 Tab 切换 / 列表+看板双视图 / 模块下事项聚合
⭐ 重点专项 — 事项类型=专项的筛选
甘特图 / 时间轴 / 列表视图 / 里程碑+风险
🎯 OKR — 支持手动导入
OKR周期管理 / 目标与关键结果 / 进度跟踪 / 手动导入/编辑
📊 运营看板 — 管理层首页
统计卡片 / 分布图 / 排行 / 异常提醒
👥 周会视图 — 按人聚合汇报
负责人列表 / 事项卡片展开 / 投屏演示+PPT翻页
📝 周报 — 自动生成
本周完成/进行中/延期/风险/下周重点 / 导出 PDF/Word/Excel
📈 数据驾驶舱 — KPI Dashboard
预算/差旅/工单/活动/满意度等指标可视化
1

数据流向：员工更新 Work Item → 系统自动聚合 → 周会视图（按人）、行政总览（按模块）、运营看板（统计）、周报（文档）、驾驶舱（KPI）

2

无冗余输入：不存在独立的"写周报"动作。周报 = 所有人本周更新的 Work Item 摘要。

3

视图即投影：每个页面只是同一份 Work Item 数据的不同筛选/排序/聚合方式。
